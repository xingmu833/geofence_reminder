**项目名称**：Geofence Reminder（临场记）

**对应 PRD**：`临场记产品需求文档（PRD）.md`

## 1. 文档目的

本文档用于把 PRD 转换为可执行的开发方案，作为 Android V1.0 MVP 的开发基线。目标是统一以下内容：

- 项目边界与阶段目标
- 工程架构与模块职责
- 数据结构与关键流程
- 开发排期、测试策略和验收口径

本文档确认后，可直接进入编码阶段。

## 2. 项目目标与范围

### 2.1 当前阶段目标

Android 端优先完成一个可发布的 MVP，围绕“进入指定地点自动提醒”这一主链路交付完整能力。

### 2.2 V1.0 必做范围

- 提醒的创建、编辑、删除、启停
- 地图选点与围栏半径设置
- 本地 SQLite 存储
- App 启动时恢复有效围栏
- 后台地理围栏监听
- 进入围栏后的本地通知提醒
- 通知点击回到对应提醒详情
- 权限引导、电池优化引导、通知权限引导

### 2.3 V1.0 暂不实现

- 云同步
- 登录/账号系统
- 日历、待办、复杂分类能力
- iOS 适配
- 蓝牙辅助触发
- 提醒规则批量导入导出

## 3. 开发原则

- 以 Android 可用性优先，避免在 MVP 阶段引入过重抽象。
- 所有位置数据、提醒规则和应用配置仅保存在本地，不接入服务端，不做账号体系。
- 功能设计优先覆盖主链路，再处理高级选项与兼容性。
- 先保证“能稳定触发”，再优化 UI 和扩展能力。
- 所有平台相关逻辑集中在 `services/` 与 `utils/`，避免散落到页面层。

### 4.3 地图方案建议

### 4.3 地图与定位方案决策

本项目面向中国大陆 Android 用户，地图方案按以下优先级执行：

1. 优先采用国内大厂的免费官方能力。
2. 若免费能力存在明显配额或商业限制，再评估是否接受。
3. 若官方 SDK 集成成本显著高于收益，再退回 `flutter_map + OSM` 作为 MVP 兜底方案。

### 4.4 当前核实结论

- 百度地图官方文档当前明确写明：Android 地图 SDK 免费对外开放，接口使用无次数限制；Android 定位 SDK 也免费对外开放，接口使用无次数限制。
- 高德开放平台当前官方定价页明确写明：基础地图定位、基础 LBS、基础搜索均采用“月配额 + 超额付费”的模式，不属于无条件永久免费。
- 因此在“优先免费”的前提下，百度地图优先级高于高德。

### 4.5 实施建议

- V1.0 优先评估百度地图 Android 原生 SDK + 百度定位 SDK 的 Flutter 接入方案。
- 若百度 Flutter 侧现成封装不足、接入复杂度明显拖慢 MVP，则退回两种备选：
  - 方案 A：高德官方 Flutter 插件或 Android SDK 接入，接受其配额模式。
  - 方案 B：`flutter_map + OSM`，仅承担基础地图展示与手动地点名修正。
- 在真正开始地图模块编码前，先做 1 次 Flutter 可集成性验证，避免选到“免费但开发代价过高”的方案。

## 7. 核心业务规则

### 7.1 提醒有效性规则

- `title` 不能为空。
- 地图必须已选点。
- `radius` 仅支持 `100 / 200 / 500 / 1000`。
- `start_time` 和 `end_time` 同时为空时视为全天有效。
- 若仅填写一个时间点，保存时直接拦截。
- 禁用状态的提醒不得注册 geofence。

### 7.2 触发规则

- 仅在“进入围栏”事件时触发。
- 若提醒处于禁用状态，则忽略事件。
- 若设置了有效时段，则只有在当前时间落入时段内才允许触发。
- 若 `trigger_limit = once_per_day`，同一自然日仅触发一次。
- 若距 `last_triggered_at` 小于 `cooldown_minutes`，则不重复触发。

### 7.3 删除与编辑规则

- 删除提醒时，必须同步移除 geofence 监听。
- 编辑提醒时，按“先移除旧围栏，再注册新围栏”的顺序处理。
- 若编辑后变为禁用状态，只保留数据，不再注册 geofence。

## 8. 核心流程设计

### 8.1 App 启动流程

1. 初始化 SQLite、通知服务、权限状态。
2. 读取本地提醒列表。
3. 筛出 `is_enabled = 1` 的提醒。
4. 启动 geofence 服务并重新注册所有有效围栏。
5. 监听通知点击事件，准备路由跳转。

### 8.2 新建提醒流程

1. 打开地图页，请求前台定位权限。
2. 获取当前位置并作为地图初始中心点。
3. 用户长按地图落点。
4. 用户选择半径，填写提醒内容，可修改地点名。
5. 点击保存。
6. 数据校验通过后写入数据库。
7. 若提醒启用，则注册 geofence。
8. 返回列表页并刷新。

### 8.3 围栏触发流程

1. 后台服务收到 enter geofence 事件。
2. 根据 geofence id 查找本地提醒记录。
3. 校验提醒是否仍启用、是否处于有效时段、是否命中去重规则。
4. 满足条件则发出本地通知。
5. 更新 `last_trigger_date` 与 `last_triggered_at`。
6. 若 App 在前台，可附加 Toast 或 Snackbar 提示。

### 8.4 通知点击流程

1. 读取通知 payload 中的 `reminderId`。
2. App 冷启动时在 bootstrap 阶段缓存跳转参数。
3. 首页加载完成后自动跳转到提醒详情页。

## 9. 核心模块设计

### 9.1 `ReminderRepository`

职责：

- 提醒增删改查
- 搜索提醒
- 更新触发状态字段
- 启停提醒状态

建议接口：

```dart
abstract class ReminderRepository {
  Future<List<Reminder>> getAll();
  Future<List<Reminder>> search(String keyword);
  Future<Reminder?> getById(int id);
  Future<int> create(Reminder reminder);
  Future<void> update(Reminder reminder);
  Future<void> delete(int id);
  Future<void> setEnabled(int id, bool enabled);
  Future<void> markTriggered(int id, DateTime now);
}
```

### 9.2 `GeofenceService`

职责：

- 初始化后台定位插件
- 注册/移除单个 geofence
- 批量恢复 geofence
- 监听 enter 事件

建议接口：

```dart
abstract class GeofenceService {
  Future<void> init();
  Future<void> start();
  Future<void> stop();
  Future<void> registerReminder(Reminder reminder);
  Future<void> unregisterReminder(int reminderId);
  Future<void> restoreAll(List<Reminder> reminders);
  Stream<int> get onReminderEntered;
}
```

### 9.3 `NotificationService`

职责：

- 初始化通知渠道
- 发送本地通知
- 解析通知点击 payload

建议接口：

```dart
abstract class NotificationService {
  Future<void> init();
  Future<void> showReminderNotification(Reminder reminder);
  Stream<int> get onNotificationTap;
}
```

### 9.4 `PermissionService`

职责：

- 查询和请求定位、后台定位、通知权限
- 为页面提供统一状态对象
- 返回是否需要展示引导页

### 9.5 `VendorSettingsService`

职责：

- 判断 ROM 厂商
- 提供电池优化、自启动、后台运行设置页跳转入口
- 若无法直达目标页，则降级到应用详情页

## 15. 风险与决策记录

### 15.1 已知风险

- Android 厂商后台限制会直接影响围栏可靠性。
- OSM 在国内网络与地点名称能力上可能不稳定。
- 背景定位权限若首次即请求，用户流失风险较高。

### 15.2 当前建议

- 首次只申请前台定位和通知权限。
- 当用户首次保存提醒或明确开启后台能力时，再请求后台定位。
- 保留手动刷新和手动查看详情的兜底路径，不把体验完全押在系统后台行为上。
- 后端与云端能力完全不纳入 V1 范围，所有设计默认单机本地运行。

### 15.3 待确认事项

- V1 是否必须交付搜索功能。
- V1 是否需要“标记完成”而不只是删除/禁用。
- 百度地图 Flutter 侧接入复杂度是否可接受。
